# openCenter CLI v1.0.0-rc02 -- Release Notes

> **Purpose:** For operators and platform engineers, lists features, fixes, breaking changes, and known limitations shipped in openCenter CLI 1.0.0-rc02.

---

## Overview

Second release candidate for openCenter CLI 1.0.0. This release delivers worker pool management, VMware/Baremetal deployment, a complete configuration system rewrite to v2, CI pipeline hardening with cosign artifact signing, and a full documentation overhaul using the Diátaxis framework.

---

## Highlights

* **Worker Pool Management** -- Add, update, scale, and remove Linux and Windows worker pools
* **VMware/Baremetal Deploy** -- `cluster deploy` now works for VMware and Baremetal providers (previously OpenStack and Kind only)
* **Configuration v2** -- Complete migration to native v2 config system; v1 types, shims, and migration code removed
* **Documentation Overhaul** -- Full Antora site structure, Diátaxis frontmatter, per-service reference pages
* **CI Signing** -- Cosign OIDC keyless signing of release artifacts
* **Dead Code Removal** -- ~3,070 lines of unused symbols eliminated

---

## New Features

### Worker Pool Management

```bash
opencenter cluster pool add <name>
opencenter cluster pool update <name>
opencenter cluster pool scale <name> --count=N
opencenter cluster pool remove <name>
opencenter cluster pool list
```

Supports both Linux and Windows worker nodes. Pool removal requires scaling to zero first.

### VMware/Baremetal Cluster Deploy

`opencenter cluster deploy` now provisions infrastructure and bootstraps Kubernetes for VMware vSphere and Baremetal providers. Includes:

* Unified `Compute` node definitions across VMware and Baremetal
* Default compute nodes for VMware provider
* Bastion SSH preflight checks
* Terraform/OpenTofu fallback (`tofu` → `terraform`)

### Flux Kustomization Improvements

* `cluster.name` injected into Flux Kustomization base template
* Gateway dependency corrected to `envoy-gateway-api-base`

### CLI Enhancements

* `cluster select` alias for `cluster use`
* Auto-commit removed from `cluster deploy` (operator controls git explicitly)

### cert-manager AWS Support

* AWS credential defaults to `us-east-1` region
* Secret uses `stringData` instead of `data` for cleaner management

## Bug Fixes

* **OpenStack CSI/CCM** -- Deployed to dedicated namespaces instead of `kube-system`
* **Gateway template** -- Renders hostnames with full ClusterFQDN
* **rmpk-gateway** -- cluster-issuer set to `letsencrypt-default`
* **VMware** -- `kubelet_rotate_server_certificates` defaults to `false`; `subnet_nodes` reads from networking CIDR; preflight reads `secrets.vsphere_csi` typed struct fields
* **OpenTofu** -- Falls back to `terraform` binary when `tofu` is not on PATH
* **Go stdlib** -- Upgraded 1.26.3 → 1.26.4 to resolve vulnerabilities

---

## Configuration System (v2 Migration Complete)

The internal configuration system has been fully rewritten:

* All v1 types, shims, re-export wrappers, and migration code removed
* `ConfigurationManager` moved to `v2` package
* `ParseClusterIdentifier` moved to `core/paths`
* Logging extracted into `internal/logging` package
* `cli_config.go` renamed to `cli_settings.go`
* ~3,070 lines of dead code and unused symbols removed

This is an internal refactor. The user-facing configuration file format is unchanged.

---

## CI/CD Improvements

* Cosign OIDC keyless signing with `id-token:write` permission
* `cosign sign-blob` updated to `--bundle` (v3 API)
* Cosign-installer pinned to v4.1.2
* GitHub Actions upgraded to Node.js 22/24 compatible versions
* Flaky property test stabilized

---

## Documentation

* Full Antora component structure with `nav.adoc` navigation
* All pages converted to Diátaxis frontmatter (`doc_type`, `audience`, `tags`)
* Per-service reference documentation for all 20+ platform services
* Stale paths repaired, dead links fixed, Purpose lines added
* Codemaps refreshed to match current codebase

---

## Breaking Changes

* **`--guided` flag removed** -- Use `opencenter cluster configure <name>` for interactive setup
* **Auto-commit removed from deploy** -- `cluster deploy` no longer commits to git automatically; run `git commit` explicitly after reviewing changes

---

## Known Limitations

* AWS provider support is planned but not yet implemented
* ArgoCD integration is in progress (FluxCD is fully supported)
* Air-gapped deployment requires the separate openCenter-AirGap package
* Windows worker node support requires the separate opencenter-windows package

---

## Upgrade from rc01

No configuration file changes required. The v2 config format is the same as rc01.

```bash
# Rebuild from source
mise run build

# Verify
./bin/opencenter version
./bin/opencenter cluster validate <name>
```
